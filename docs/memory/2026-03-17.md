# 银行招聘岗位爬取实现计划 (2026-03-17)

## 目标
在 JobRadar 项目内实现银行招聘岗位爬取能力，覆盖三类银行：
1. 大型银行（工农中建交邮储 + 国开/进出口/农发行）
2. 江浙沪城商行（南京银行、宁波银行、杭州银行、江苏银行、上海银行、苏州银行、杭州联合等）
3. 股份制银行（招行、浦发、中信、民生、兴业、平安、光大、华夏、广发、浙商、渤海、恒丰等）

## 设计思路（借鉴 MediaCrawler）
- 模块化爬虫：每个银行一个独立爬虫模块
- 统一配置管理：使用 YAML/JSON 配置站点列表和爬取规则
- 统一入口脚本：scripts/run_bank_crawl.py
- 数据流统一：写入 JobRadar 现有数据库和 CSV
- 去重机制：基于 job_id + source

## 实施步骤

### 1. 创建银行爬虫模块结构
- 创建 `JobRadar/backend/app/services/bank_crawler/` 目录
- 实现 base_crawler.py（基础爬虫类）
- 实现各银行爬虫（首批 5-10 家样本）

### 2. 配置管理
- 创建银行站点配置文件 `bank_sites.yaml`
- 包含：站点 URL、API 接口、选择器规则、字段映射等

### 3. 数据模型适配
- 复用 Job 模型（已有 job_id, source, company, job_title 等字段）
- source 设置为 "bank-xxx"（如 "bank-icbc"）

### 4. 去重机制
- 基于 job_id + source 的唯一性
- 使用现有 DB 去重逻辑

### 5. 首批目标银行（5-10 家样本）
优先选择有公开招聘网站的银行：
- 招商银行（招聘中心完善）
- 浦发银行（招聘中心完善）
- 宁波银行（江浙沪城商行代表）
- 江苏银行（江浙沪城商行代表）
- 中国银行（大型银行代表）
- 建设银行（大型银行代表）

### 6. 技术选型
- 优先使用 requests + BeautifulSoup（轻量、稳定）
- 部分网站可能需要 Playwright（动态渲染）
- 数据存储：SQLite（现有 JobRadar 数据库）
- 日志：CrawlLog 表（现有）

### 7. 输出
- scripts/run_bank_crawl.py（统一入口）
- 银行爬虫使用文档
- 扩展站点配置指南
- 常见故障排查
