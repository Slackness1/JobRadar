- 一次接口验证中出现过：

  - 但直接查询 SQLite 某次显示 `job_intel_records` / `job_intel_snapshots` 为 0
- 这说明当时可能存在：
  - 不同进程/不同连接查看时机差异，或
  - 启动方式造成的 SQLite 可见性/进程状态差异
- 后续若继续推进真实接入，建议先统一后端启动方式，再复核一遍 records/snapshots 的持久化一致性。
