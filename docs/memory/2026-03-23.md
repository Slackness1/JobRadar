### 本机前端“看不到数据”问题的真实原因
- 2026-03-23 晚间排查确认：不是数据库丢失，而是 **前端还在跑、后端服务挂了**。
- JobRadar 后端实际主库是：`JobRadar/backend/data/jobradar.db`
- `JobRadar/backend/data/jobs.db` 是旧的兼容文件，当前为 0 字节，不是主库。
- 当晚真实库内岗位数约为 **22979**；前端空白时是因为 `127.0.0.1:8000` 后端未监听。
- 恢复方式：使用 `JobRadar/backend/.venv` 启动 uvicorn，日志落到 `data/tmp/jobradar_backend.log`。
- `/api/jobs/stats` 返回的展示量可能小于库内总量，因为接口层存在 `get_spring_display_cutoff` 之类展示过滤，不应误判为“数据丢失”。

### Moka 家族可复用提取链路（已验证可落地）
- 页面真实入口里可从 `init-data` / 页面初始化数据中拿到 `siteId`、`orgId`、`aesIv`。
- 列表接口：`POST https://app.mokahr.com/api/outer/ats-apply/website/jobs/v2`
- 详情接口：`POST https://app.mokahr.com/api/outer/ats-apply/website/job`
- 返回体需使用 `data + necromancer + aesIv` 做 **AES-CBC + PKCS7** 解密后再解析 JSON。
- 此链路已成功用于：
  - 天合光能（40）
  - 远景能源（23）

### 阿特斯（CSI Solar）自建 SPA 可复用接口

  - 列表：`POST /api/campus-recruit/common/recruit/allPosition`
  - 详情：`GET /api/campus-recruit/common/recruit/positionDetail/{id}`
- 列表分页参数已验证：`current`、`pageSize`、`status=1`
- 此链路已成功正式提取并入库 **208 条**。

### 上海电气 Hotjob 家族可复用入口与列表接口
